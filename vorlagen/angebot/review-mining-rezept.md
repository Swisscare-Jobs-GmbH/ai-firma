# Review-Mining — Kurz-Rezept fuer belegte Angebots-Zahlen

> Zweck: Der Stolz-Einstieg im `momente-angebot.html` steht und faellt mit BELEGTEN Zahlen. Eine
> erfundene oder aus dem Kopf geschaetzte Zahl ("ueber 1000 Bewertungen") ist die Fehlerklasse, die
> das ganze Blatt kippt. Darum: vor jedem Angebot live nachzaehlen.
>
> Muster-Beleg: GzF-Schmerz-Fragen (Abschnitt "Live-Beleg Google Maps", 19.07) + Finelli-Reviews-Doc.
> Dauer: ~15-20 Min pro Kunde.

## Fazit-Regel

**Keine belegte Zahl → Satz weglassen, nie schaetzen.** Lieber eine Zahl weniger als eine falsche.
Wenn der Kunde eine hoehere Zahl im Kopf hat als du findest: seine Quelle klaeren, nicht uebernehmen.

## Rezept (4 Schritte)

### 1. Google Maps live nachzaehlen (eingeloggt, im Browser)
- Kunden-Eintrag oeffnen (Maps-Link notieren, z.B. `https://maps.google.com/?cid={CID}`).
- **Sterne-Schnitt + Gesamtzahl** ablesen.
- **Verteilung notieren:** wie viele 5★ / 4★ / 3★ / 2★ / 1★. Beispiel-Notation:
  `4,6 Sterne, 22 Bewertungen — 17x 5★ / 3x 4★ / 1x 3★ / 0x 2★ / 1x 1★ → 20 von 22 = 4-5★`.
- Googles eigene **Themen-Chips** (z.B. "lieferung 10x", "beratung 3x") mitnehmen — sie zeigen,
  worueber die Kunden von selbst reden.

### 2. Weitere Plattformen dazu (falls vorhanden)
- Trustpilot, Branchen-Portale, Social. Je Plattform: Anzahl + Note getrennt notieren.
- **Gesamt-Zahl transparent zusammensetzen** ("130 Google + 41 Trustpilot = ueber 170"), nie eine
  glatte Marketing-Zahl bilden.
- Geloeschte Reviews sind technisch nicht abrufbar — ehrlich so benennen, nicht versprechen.

### 3. Top-Lob-Themen clustern (fuers Anerkennen im Einstieg)
- 3-5 Themen mit je ~Nennungs-Zahl: `Beratung (~3x)`, `Material-Qualitaet (~10x)`, `Service (~5x)`.
- 1-2 woertliche Lob-Zitate rausziehen (kurz, mit Quelle) — die kommen in die Intro.

### 4. Kritik-Themen clustern (die Angebots-Munition)
- Negativ-Themen zaehlen und clustern — **das sind die Schmerzen, die deine Software lindert.**
- Vor dem Termin (2 Min, eingeloggt): Rezensionen → "Niedrigste" → Text der 1★/3★ lesen.
- Pro Kritik-Cluster einen Satz "was unsere Software direkt lindert" formulieren (belegbar halten).
