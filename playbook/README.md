# Playbook: Neuer Software-Kunde

> Zweck: dasselbe Vorgehen wie bei Finelli — **schneller, mit weniger Fragen, automatischer.**
> Destilliert 18.-19.07.2026. Jede Phase hat ein eigenes Blatt (`phase-0` … `phase-5`).
> Pflicht-Haltung fuers Verkaufen: nie negativ eroeffnen, Hormozi-Reihenfolge (siehe Phase 4).

## Die 6 Phasen (Finelli-erprobt)

| Phase | Was | Ergebnis | Dauer |
|---|---|---|---|
| **0 — Recherche** | Firmen-Profil (Handelsregister + Web), ALLE Reviews LIVE, Schmerz-Hypothesen | Profil-Blatt + Hypothesen | 0.5 Tag |
| **1 — Schmerz-Landkarte** | LIVE mit Kunde: 10 Schmerz-Fragen, jede Antwort woertlich + mit Zahlen | Schmerz-Tabelle + Killer-Pain | 1 Gespraech |
| **2 — Deal** | Wert-Ranking (Hormozi) → Deal-Bausteine + Vertrag | Angebots-Zahlen + Vertrags-Entwurf | 0.5 Tag |
| **3 — MVP bauen** | Uebungs-Modus zuerst (Mock), Klick-Beweis, Review-Schwarm + Fixes | Demo-faehige App | 1-3 Tage |
| **4 — Angebot** | Hormozi-Reihenfolge, Stolz-Einstieg, Judge-Panel drueber | 1 Seite A4 (PDF) | 0.5 Tag |
| **5 — Termin** | Demo aus laufender App, Kunde klickt selbst, Abnahme-Liste, Vertrag | Unterschrift + Zugaenge | 1 Besuch |

## Reihenfolge-Regel (wichtig)

**Erst Angebot wasserdicht, DANN Fragen schicken + bauen.** Der Kunde muss verstehen, was er
unterschreibt, bevor Zeit in den Bau fliesst (SA-Direktive 18.07). Und: **Verkauf vor Bau** —
Leitplanke 1 aus `CLAUDE.md`.

## Automatisierungs-Hebel fuer Kunde Nr. 2+

- **Repo-Start:** `finelli-cockpit` klonen, Mock-Daten + Branding tauschen (~0.5 Tag statt 2).
- **Phase 0 + 4** laufen als Agenten-Workflows (Recherche-Schwarm, Judge-Panel).
- **Vertrag + Angebot:** nur Zahlen/Namen/Momente tauschen, Struktur steht (siehe `vorlagen/`).
